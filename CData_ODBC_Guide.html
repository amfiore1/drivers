1. Introduction

CData ODBC Drivers let you connect SaaS, NoSQL, and big-data sources to ODBC-compliant tools (Excel, Power BI, Tableau) and programming languages (C#, Python, R). They expose your source as relational tables via SQL.

Tip: The same connection and query patterns apply to all CData ODBC drivers—only the connection properties change per source.

2. Installation

  1. Download the CData ODBC Driver for your source.
  2. Run the installer (choose the 64‑bit build for modern BI tools).
  3. Restart your ODBC tool if it was open during installation.

3. Create a DSN

  1. Open ODBC Data Sources (64‑bit).
  2. Go to System DSN → Add.
  3. Select your CData <Source> ODBC Driver.
  4. Enter authentication details and test the connection.

DSN‑less alternative: embed all properties directly in the connection string (see examples below).

4. Using in Tools

  • Excel: Data → Get Data → From ODBC → Select DSN.
  • Power BI: Home → Get Data → ODBC → Select DSN.
  • Tableau: Connect → To a Server → Other Databases (ODBC) → DSN.

5. Code Examples

C#

using System.Data.Odbc;
var conn = new OdbcConnection("DSN=CDataSalesforce");
conn.Open();
var cmd = new OdbcCommand("SELECT Name FROM Account", conn);
using var rdr = cmd.ExecuteReader();
while (rdr.Read()) Console.WriteLine(rdr["Name"]);

Python (pyodbc)

import pyodbc
conn = pyodbc.connect("DSN=CDataSalesforce")
cur = conn.cursor()
cur.execute("SELECT Name FROM Account")
for row in cur.fetchall():
    print(row.Name)

R

library(DBI)
con <- dbConnect(odbc::odbc(), "CDataSalesforce")
dbGetQuery(con, "SELECT Name FROM Account")

6. Troubleshooting

  • DSN not found → Ensure a 64‑bit System DSN and tool bitness matches (64‑bit Excel/Power BI).
  • Auth fails → Verify connection properties (OAuth tokens, API keys, domains).
  • Slow queries → Use WHERE filters, select only needed columns, enable caching if supported.

7. Best Practices

  • Use System DSNs for shared environments.
  • Filter and aggregate with SQL to push work down to the source.
  • Parameterize queries for reusability and security.
  • Document source-specific connection properties in an appendix.
